## How To Resume On The New Computer

Use the full repository archive if you want the simplest path. It includes the `.git` directory and the current working tree.

```bash
mkdir -p ~/Projects
cd ~/Projects
tar -xzf /path/to/felix-alerts-full-repo-YYYYMMDD-HHMMSS.tar.gz
cd "felix alerts"
git status --short
```

Then commit the handoff changes:

```bash
PYTHONPATH=backend:. python3 -m unittest discover -s tests
python3 scripts/validate_upload_package.py
git add backend frontend deploy docs tests
git commit -m "feat: production readiness and alert status cleanup"
```

Push to GitHub:

```bash
git remote -v
git push -u origin main
```

If the remote is missing on the new computer:

```bash
git remote add origin https://github.com/meiriohad76-oss/felix-alerts.git
git push -u origin main
```

## Raspberry Pi Deployment

Use the included deployment archive:

```bash
scp sentinel-deploy-20260520-163854.tar.gz ahad@<PI_HOST_OR_IP>:/tmp/
ssh ahad@<PI_HOST_OR_IP>
```

On the Pi:

```bash
sudo mkdir -p /var/backups/sentinel
sudo systemctl stop sentinel
sudo cp /var/lib/sentinel/sentinel.sqlite3 "/var/backups/sentinel/sentinel-predeploy-$(date -u +%Y%m%d-%H%M%S).sqlite3"
sudo systemctl start sentinel

sudo systemctl stop sentinel
sudo mkdir -p /opt/sentinel
sudo tar -xzf /tmp/sentinel-deploy-20260520-163854.tar.gz -C /opt/sentinel
sudo bash /opt/sentinel/deploy/raspberry-pi/install.sh
sudo systemctl restart sentinel
```

Verify:

```bash
curl -i http://127.0.0.1:8765/health
curl -i https://sentinel1.ahaddashboards.uk/health
sudo journalctl -u sentinel -n 120 --no-pager
```
